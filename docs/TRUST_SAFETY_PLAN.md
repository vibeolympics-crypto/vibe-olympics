# Vibe Olympics 신뢰 & 안전 시스템 기획서 v2

> **철학 변경**: "선 신뢰, 후 검증" (Trust First, Verify Later)
>
> 모든 사용자를 잠재적 위반자로 보지 않고, 선량한 참여자로 먼저 신뢰합니다.
> 문제를 일으킨 사람에게만 제약을 적용합니다.

---

## 핵심 원칙

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   ❌ 기존 접근 (빡빡한 진입 장벽)                               │
│   "증명하기 전까지는 의심한다"                                  │
│   → 신규 판매자 이탈, 성장 저해                                 │
│                                                                 │
│   ✅ 새로운 접근 (열린 진입, 선별적 제재)                       │
│   "문제가 생기기 전까지는 신뢰한다"                             │
│   → 쉬운 시작, 빠른 성장, 문제 발생 시 대응                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 1. 판매자 가입 및 상품 등록 (간소화)

### 1.1 최소 요건 (누구나 시작 가능)

```
판매자 등록 요건:
├── 이메일 인증 ✓
├── 휴대폰 인증 ✓ (본인 확인용)
└── 이용약관 동의 ✓

상품 등록 요건:
├── 제목 (2자 이상)
├── 설명 (50자 이상)
├── 가격 설정
├── 파일 업로드
└── 완료! 🎉
```

### 1.2 자동 검수는 최소한으로

| 검수 항목 | 적용 여부 | 이유 |
|----------|----------|------|
| 금칙어 필터 | ✅ 적용 | 명백한 불법/유해 콘텐츠만 |
| 악성코드 검사 | ✅ 적용 | 안전 필수 사항 |
| AI 품질 검사 | ❌ 미적용 | 진입 장벽 제거 |
| 수동 승인 | ❌ 미적용 | 즉시 판매 가능 |

### 1.3 첫 판매자 환영 메시지

```
🎉 축하합니다! 첫 상품이 등록되었습니다.

Vibe Olympics는 창작자를 응원합니다.
별도의 복잡한 인증 없이 바로 판매를 시작하실 수 있습니다.

💡 Tip: 상세한 설명과 미리보기를 제공하면 판매에 도움이 됩니다.

문의사항이 있으시면 언제든 고객센터로 연락주세요.
좋은 거래 되시길 바랍니다! 🙌
```

## 2. 제약 적용 기준 (문제 발생 시에만)

### 2.1 단계별 제약 시스템

```
┌─────────────────────────────────────────────────────────────────┐
│  🟢 정상 상태 (기본)                                            │
│  - 모든 기능 이용 가능                                          │
│  - 상품 등록 즉시 게시                                          │
│  - 정산 주기: 7일                                               │
└─────────────────────────────────────────────────────────────────┘
                              │
                    (신고 접수 또는 분쟁 발생)
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  🟡 주의 상태 (1회 문제 발생)                                   │
│  - 기능 제한 없음                                               │
│  - 해당 건 모니터링                                             │
│  - 안내 메시지 발송                                             │
│  "신고가 접수되었습니다. 확인 중이며, 문제없으면 해제됩니다."   │
└─────────────────────────────────────────────────────────────────┘
                              │
                    (2회 이상 경고 또는 피해 확정)
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  🟠 제한 상태                                                   │
│  - 신규 상품 등록 시 검수 필요 (24시간 내 승인)                 │
│  - 정산 주기: 14일로 연장                                       │
│  - 제한 사유 및 해제 조건 안내                                  │
└─────────────────────────────────────────────────────────────────┘
                              │
                    (3회 이상 경고 또는 심각한 피해)
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  🔴 정지 상태                                                   │
│  - 판매 활동 일시 중지                                          │
│  - 기존 수익 보류                                               │
│  - 이의 신청 가능 (7일 내)                                      │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 제약 적용 기준 (명확한 기준)

| 상황 | 조치 | 회복 조건 |
|------|------|----------|
| 첫 번째 신고 (경미) | 🟢 유지 + 모니터링 | 자동 |
| 두 번째 신고 | 🟡 주의 | 30일 무사고 |
| 피해 확정 1회 | 🟠 제한 | 60일 무사고 + 피해 보상 |
| 피해 확정 2회 | 🔴 정지 | 이의 신청 심사 |
| 명백한 사기 | ⛔ 영구 정지 | 불가 |

### 2.3 "이력자" 판별 기준

```typescript
interface RestrictionCriteria {
  // 제약 적용 대상 (OR 조건)
  shouldRestrict: {
    confirmedFraud: boolean;           // 사기 확정 이력
    confirmedDamage: boolean;          // 피해 발생 확정
    warningCount: number >= 2;         // 경고 2회 이상
    refundRate: number > 0.3;          // 환불률 30% 초과
    disputeLossCount: number >= 2;     // 분쟁 패소 2회 이상
  };

  // 제약 미적용 대상 (대부분의 사용자)
  normalUser: {
    warningCount: 0 | 1;               // 경고 0-1회
    noConfirmedDamage: true;           // 피해 발생 없음
    normalRefundRate: true;            // 정상 환불률
  };
}
```

## 3. 구매자 보호 (피해 발생 후 대응)

### 3.1 기본 보호 장치 (모든 거래)

```
모든 거래에 기본 적용:
├── 7일 환불 보장 (다운로드 전)
├── 72시간 문의 응답 보장
├── 분쟁 신청 권리
└── 고객센터 지원
```

### 3.2 환불 정책 (관대한 정책)

| 상황 | 환불 | 판매자 제재 |
|------|------|------------|
| 다운로드 전 취소 | ✅ 즉시 환불 | ❌ 없음 |
| 상품 불량/미작동 | ✅ 검토 후 환불 | 🟡 사실 확인 |
| 설명과 다른 상품 | ✅ 검토 후 환불 | 🟡 사실 확인 |
| 단순 변심 (다운로드 후) | ❌ 불가 | ❌ 없음 |

### 3.3 분쟁 시 중립적 접근

```
분쟁 발생 시:

1. 양측 의견 청취 (각각 48시간)
2. 증거 자료 검토
3. 중립적 판정

원칙:
- 구매자/판매자 어느 쪽도 일방적으로 유리하지 않음
- 증거 기반 판단
- 명확하지 않으면 양측 협의 유도
```

## 4. 신뢰 구축 시스템 (자발적 인센티브)

### 4.1 배지 시스템 (강제 아닌 혜택)

```
배지는 제약이 아닌 혜택입니다.
배지가 없어도 판매에 제한은 없습니다.

🌱 신규 판매자
   └── 기본 (누구나)

✅ 인증 판매자
   └── 조건: 판매 10건 + 평점 4.0+
   └── 혜택: 인증 배지 표시, 검색 노출 +10%

⭐ 우수 판매자
   └── 조건: 판매 50건 + 평점 4.5+ + 환불률 5% 미만
   └── 혜택: 우수 배지, 검색 노출 +20%, 정산 주기 단축

💎 프리미엄 판매자
   └── 조건: 판매 200건 + 평점 4.8+ + 환불률 2% 미만
   └── 혜택: 프리미엄 배지, 홈 노출, 수수료 할인
```

### 4.2 자연스러운 신뢰 지표

| 지표 | 표시 위치 | 목적 |
|------|----------|------|
| 판매 건수 | 판매자 프로필 | 경험 수준 |
| 평균 평점 | 상품 페이지 | 품질 참고 |
| 응답률 | 판매자 프로필 | 소통 능력 |
| 가입 기간 | 판매자 프로필 | 신뢰도 |

```
이 지표들은 자동으로 계산되며,
판매자가 별도로 인증하거나 신청할 필요 없습니다.
좋은 활동이 자연스럽게 반영됩니다.
```

## 5. 신고 시스템 (피해 발생 대응)

### 5.1 신고 처리 원칙

```
신고 접수 시:

1. 즉시 삭제하지 않음 (무고 방지)
2. 피신고인에게 소명 기회 부여
3. 증거 기반 판단
4. 명백한 경우에만 즉시 조치
```

### 5.2 신고 유형별 처리

| 유형 | 초기 조치 | 확정 시 조치 |
|------|----------|-------------|
| 저작권 침해 | 임시 비공개 | 삭제 + 경고 |
| 사기 의심 | 모니터링 | 정지 + 환불 |
| 불량 상품 | 모니터링 | 경고 |
| 욕설/비방 | 숨김 처리 | 삭제 + 경고 |

### 5.3 악의적 신고 방지

```typescript
// 신고자도 평가됩니다
interface ReporterReputation {
  totalReports: number;
  validReports: number;      // 인정된 신고
  invalidReports: number;    // 기각된 신고

  // 허위 신고 비율이 높으면
  // 신고 우선순위 하락
  reliability: number;  // validReports / totalReports
}
```

```
      ▼

      │
```

### 6.2 피해자 보호 기금 (선택적 운영)

```
플랫폼이 직접 피해를 보상하지는 않지만,
명백한 사기 피해의 경우:

1. 판매자 보류금에서 우선 환불
2. 보류금 부족 시 플랫폼 기금에서 일부 지원 검토
3. 법적 조치 지원 (증거 제공)
```

## 7. 성장 단계별 정책 조정

### Phase 1: 성장기 (현재)

```
목표: 판매자 유입, 거래량 증가

정책:
├── 최소한의 진입 장벽
├── 관대한 환불 정책
├── 신고 발생 시에만 대응
├── 수동 검수 최소화
└── 판매자 친화적 정산
```

### Phase 2: 안정기 (거래 1,000건+)

```
목표: 품질 향상, 신뢰 구축

정책:
├── 리뷰 시스템 강화
├── 배지 시스템 활성화
├── 자동 모니터링 도입
└── 우수 판매자 혜택 강화
```

### Phase 3: 성숙기 (거래 10,000건+)

```
목표: 프리미엄 마켓플레이스

정책:
├── AI 기반 품질 관리
├── 고급 사기 탐지
├── 판매자 등급제 본격화
└── 프리미엄 서비스 도입
```

## 8. 기존 안 vs 수정안 비교

| 항목 | 기존안 (v1) | 수정안 (v2) |
|------|------------|------------|
| **진입 장벽** | 높음 (인증 필수) | 낮음 (이메일/휴대폰만) |
| **상품 등록** | 검수 후 게시 | 즉시 게시 |
| **신규 판매자** | 기능 제한 | 기능 제한 없음 |
| **제약 적용** | 모든 신규자 | 문제 발생자만 |
| **정산 주기** | 등급별 차등 | 기본 7일 (제재 시 연장) |
| **검수** | 자동 + 수동 | 금칙어/악성코드만 |
| **철학** | 증명 후 신뢰 | 신뢰 후 검증 |

## 10. 운영팀 가이드라인

### 10.1 기본 원칙

```
1. 의심만으로 제재하지 않는다
2. 명확한 증거 없이 판매자 불이익을 주지 않는다
3. 구매자 주장만으로 환불하지 않는다
4. 양측에 공정한 소명 기회를 준다
5. 제재 시 명확한 사유와 회복 조건을 안내한다
```

### 10.2 판단 기준

| 상황 | 판단 | 근거 |
|------|------|------|
| 첫 신고, 증거 부족 | 보류 | 무고 가능성 |
| 반복 신고, 패턴 일치 | 조치 | 문제 명확 |
| 양측 주장 상충 | 협의 유도 | 중립 유지 |
| 명백한 사기 | 즉시 정지 | 피해 방지 |

## 결론

```
🌱 성장기 마켓플레이스의 핵심:

"문 앞에서 막지 말고, 문제 생기면 대응하자"

- 99%의 선량한 사용자를 위해 문을 열어두고
- 1%의 문제 발생자에게만 제재를 적용합니다

잔디 구장 비유로:
- 모든 선수에게 보호 장비를 강제하지 않고
- 위험한 플레이를 하는 선수에게만 퇴장을 명합니다
```
