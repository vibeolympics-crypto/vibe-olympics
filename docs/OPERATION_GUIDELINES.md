```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   "선 신뢰, 후 검증" (Trust First, Verify Later)           │
│                                                             │

```

### 1.2 공정성 원칙

| 원칙 | 설명 |
|------|------|
| **무죄 추정** | 증거가 확보되기 전까지 위반으로 간주하지 않음 |
| **양측 청문** | 모든 당사자에게 소명 기회 부여 |

모든 사용자는 다음의 권리를 가집니다:

- ✅ 공정한 대우를 받을 권리
- ✅ 신고 및 분쟁 신청 권리
- ✅ 소명 기회를 가질 권리
- ✅ 제재에 대해 이의 신청할 권리
- ✅ 처리 결과를 통보받을 권리
- ✅ 개인정보 보호를 받을 권리

## 2. 사용자 상태 및 등급

### 2.1 사용자 상태 정의

#### 🟢 정상 (NORMAL)

```
상태: 모든 기능 이용 가능
조건: 기본 상태 (신규 가입 시 자동 부여)

권한:
├── 상품 등록: 즉시 게시
├── 거래: 제한 없음
├── 정산: 7일 주기
└── 기타: 모든 기능 이용 가능
```

#### 🟡 주의 (CAUTION)

```
상태: 모니터링 대상 (기능 제한 없음)
조건: 신고 접수 또는 경미한 문제 1회 발생

권한:
├── 상품 등록: 제한 없음
├── 거래: 제한 없음
├── 정산: 7일 주기 (유지)
└── 해제: 30일 무사고 시 자동 해제

안내 메시지:
"신고가 접수되어 확인 중입니다.
 검토 결과 문제가 없으면 자동으로 해제됩니다."
```

#### 🟠 제한 (RESTRICTED)

```
상태: 일부 기능 제한
조건: 경고 2회 이상 또는 피해 확정 1회

권한:
├── 상품 등록: 검수 후 게시 (24시간 내)
├── 거래: 제한 없음
├── 정산: 14일 주기로 연장
└── 해제: 60일 무사고 + 피해 보상 완료

안내 메시지:
"계정이 제한 상태입니다.
 사유: [제재 사유]
 해제 조건: [조건 상세]"
```

#### 🔴 정지 (SUSPENDED)

```
상태: 판매 활동 중지
조건: 경고 3회 이상 또는 심각한 피해 발생

권한:
├── 상품 등록: 불가
├── 거래: 신규 거래 불가 (기존 거래 처리만)
├── 정산: 보류 (조사 완료 후 처리)
└── 해제: 이의 신청 심사 통과

안내 메시지:
"계정이 정지되었습니다.
 사유: [제재 사유]
 이의 신청: 7일 이내 가능"
```

#### ⛔ 영구 정지 (BANNED)

```
상태: 계정 영구 이용 불가
조건: 명백한 사기, 반복적 악의적 행위

권한:
├── 모든 기능: 이용 불가
├── 재가입: 제한
└── 미정산금: 피해 보상 후 잔액 지급

안내 메시지:
"계정이 영구 정지되었습니다.
 사유: [제재 사유]
 문의: support@vibeolympics.com"
```

## 3. 신고 시스템

### 3.1 신고 가능 유형

| 코드 | 유형 | 설명 | 처리 시한 |
|------|------|------|----------|
| `FRAUD` | 사기/기만 | 허위 상품, 미제공, 의도적 기만 | 24시간 |
| `COPYRIGHT` | 저작권 침해 | 무단 복제, 도용, 표절 | 48시간 |
| `INAPPROPRIATE` | 부적절 콘텐츠 | 불법, 유해, 성인물 | 24시간 |
| `HARASSMENT` | 괴롭힘/욕설 | 비방, 협박, 모욕 | 24시간 |
| `SPAM` | 스팸/광고 | 무분별 홍보, 도배 | 48시간 |
| `DIRECT_TRADE` | 직거래 유도 | 플랫폼 우회 거래 시도 | 48시간 |
| `QUALITY` | 품질 문제 | 설명과 다른 상품, 불량 | 72시간 |
| `OTHER` | 기타 | 위 항목에 해당하지 않는 경우 | 72시간 |

### 3.2 신고 접수 요건

```typescript
// 신고 시 필수 정보
interface ReportSubmission {
  targetType: 'USER' | 'PRODUCT' | 'REVIEW' | 'POST' | 'COMMENT';
  targetId: string;           // 신고 대상 ID
  type: ReportType;           // 신고 유형
  reason: string;             // 신고 사유 (최소 20자)
  evidence?: string[];        // 증거 자료 (스크린샷, 파일 등)
}
```

### 3.4 신고 상태

| 상태 | 코드 | 설명 |
|------|------|------|
| 접수 대기 | `PENDING` | 신고 접수됨, 검토 대기 |
| 검토 중 | `REVIEWING` | 담당자가 검토 중 |
| 소명 대기 | `AWAITING_RESPONSE` | 피신고인 소명 대기 |
| 처리 완료 | `RESOLVED` | 신고 인정, 조치 완료 |
| 기각 | `DISMISSED` | 신고 불인정 |
| 보류 | `ON_HOLD` | 추가 조사 필요 |

### 3.5 악성 신고 방지

```
신고자 신뢰도 시스템:

• 모든 신고자의 신고 이력이 기록됩니다
• 인정된 신고: 신뢰도 +1
• 기각된 신고: 신뢰도 -1
• 악의적 신고 확인: 신뢰도 -5 + 경고

신뢰도에 따른 처리:
• 높은 신뢰도: 우선 처리
• 낮은 신뢰도: 신중 검토
• 매우 낮은 신뢰도: 신고 기능 제한
```

## 4. 분쟁 해결 절차

### 4.1 분쟁 신청 요건

```
분쟁 신청 가능 조건:
├── 실제 거래가 완료된 경우
├── 거래일로부터 14일 이내
├── 당사자 간 협의 시도 후
└── 구체적인 분쟁 사유 명시

분쟁 신청 불가:
├── 거래 사실이 없는 경우
├── 이미 환불 완료된 거래
├── 분쟁 기간 경과 (14일 초과)
└── 동일 건 재신청 (결과 확정 후)
```

### 4.2 분쟁 유형

| 코드 | 유형 | 설명 | 일반 해결 방향 |
|------|------|------|---------------|
| `NOT_AS_DESCRIBED` | 상품 불일치 | 설명과 실제 상품이 다름 | 증거 기반 판단 |
| `NOT_DELIVERED` | 미제공 | 결제 후 상품 미제공 | 환불 우선 |
| `QUALITY_ISSUE` | 품질 문제 | 정상 작동 불가 | 기술적 검토 |
| `REFUND_DISPUTE` | 환불 분쟁 | 환불 요청 거부 | 정책 기준 판단 |
| `OTHER` | 기타 | 위 항목 외 분쟁 | 개별 검토 |

### 4.4 분쟁 판정 기준

| 상황 | 판정 방향 | 근거 |
|------|----------|------|
| 상품 미제공 (증거 있음) | 구매자 승 | 계약 불이행 |
| 상품 불일치 (명확한 차이) | 구매자 승 | 허위 광고 |
| 정상 상품 + 단순 변심 | 판매자 승 | 환불 정책 |

| 기술적 문제 (판매자 귀책) | 구매자 승 | 품질 보증 |
| 증거 불충분 | 기각/협의 유도 | 입증 책임 |

## 5. 제재 기준 및 절차

### 5.1 제재 유형

| 유형 | 코드 | 설명 | 기간 |
|------|------|------|------|
| 경고 | `WARNING` | 위반 사실 통지, 기능 제한 없음 | - |
| 콘텐츠 삭제 | `CONTENT_REMOVAL` | 위반 콘텐츠 삭제 | - |
| 기능 제한 | `FEATURE_RESTRICTION` | 특정 기능 이용 제한 | 7-30일 |
| 일시 정지 | `TEMPORARY_SUSPENSION` | 계정 일시 정지 | 7-90일 |

### 5.2 위반 행위별 제재 기준

## 7. 운영팀 행동 강령

### 7.1 기본 원칙

```
운영팀은 다음 원칙을 준수합니다:

1. 공정성
   • 모든 사용자를 동등하게 대우
   • 개인적 감정이나 편견 배제
   • 일관된 기준 적용

2. 중립성
   • 특정 당사자 편들지 않음
   • 증거 기반 판단
   • 양측 의견 균형있게 청취

3. 신속성
   • 정해진 시한 내 처리
   • 긴급 사안 우선 처리
   • 지연 시 사유 안내

4. 투명성
   • 처리 결과 명확히 안내
   • 판단 근거 제시
   • 이의 신청 방법 안내

5. 비밀유지
   • 개인정보 보호
   • 분쟁 내용 외부 미공개
   • 필요 최소한의 정보만 공유
```

### 7.2 금지 행위

```
운영팀원은 다음 행위가 금지됩니다:

❌ 개인적 이익을 위한 판단
❌ 증거 없이 일방 주장만으로 제재
❌ 처리 시한 무단 초과
❌ 사용자 개인정보 외부 유출
❌ 사적인 연락이나 거래
❌ 특정 사용자 우대/차별
```

### 7.3 판단 가이드라인

| 상황 | 권장 판단 | 주의사항 |
|------|----------|----------|
| 증거 충분, 위반 명확 | 즉시 조치 | 소명 기회 필수 |
| 증거 부족, 위반 의심 | 추가 조사 | 섣부른 제재 금지 |
| 양측 주장 상충 | 협의 유도 | 강제 판정 최소화 |
| 긴급 상황 (사기 등) | 임시 조치 | 사후 정식 절차 |

## 8. 용어 정의

| 용어 | 정의 |
|------|------|
| **신고** | 다른 사용자의 위반 행위를 운영팀에 알리는 행위 |
| **분쟁** | 거래 당사자 간 발생한 다툼으로, 제3자 개입이 필요한 상황 |
| **제재** | 위반 행위에 대해 플랫폼이 취하는 불이익 조치 |
| **경고** | 위반 사실을 알리고 재발 방지를 요청하는 조치 |
| **정지** | 계정의 일부 또는 전체 기능 이용을 차단하는 조치 |
| **피신고인** | 신고를 당한 사용자 |
| **신고자** | 신고를 접수한 사용자 |
| **중재** | 분쟁 당사자 간 합의를 돕는 운영팀의 개입 |
| **판정** | 분쟁에 대한 운영팀의 최종 결정 |
| **이의 신청** | 제재 결정에 대해 재검토를 요청하는 행위 |
| **소명** | 위반 의심에 대해 해명하는 행위 |
